# 绘心 品牌与 README / Logo 调整设计

## 背景

当前仓库已经完成 `Tauri + React` 启动器模板与仿绘世风格界面复刻，但品牌表达仍然偏模板默认态：

- 仓库根目录还没有项目级 `README.md`
- 应用左上角品牌区仍不是最终品牌表达
- 现有桌面图标包仍是模板阶段资源

这次调整不做功能扩展，目标是把模板仓库的对外表达先统一到一个明确品牌方向上。

## 目标

本次调整要同时解决两件事：

1. 给仓库建立清晰的对外介绍
2. 让应用内品牌表达与 README 保持一致

品牌方向确定为：`绘心`

## 已确认决策

### 品牌层级

- 对外品牌名：`绘心`
- 仓库角色：`XnneHangLab` 体系下可复用的启动器模板
- 后续产品关系：`绘心` 可以继续扩展为 `绘心-voice` 等子产品

### README 口径

- README 以中文为主
- README 主标题直接使用 `绘心`
- README 首段不把仓库写成单一业务项目，而是写成：
  - 这是一个可复用的 `XnneHangLab` 启动器模板
  - 当前以“绘心”作为品牌方向
  - 当前界面风格参考绘世启动器
  - 方向上强调温度感、陪伴感和长期演进

### 文案方向

README 首页文案使用自己的品牌表达，不直接写“像 Neuro-sama 那样”这类对外依附型描述。

可采用的核心表述是：

- `让 AI 更有温度，也更适合长期陪伴。`
- `面向陪伴感、持续成长与长期使用的桌面启动器模板。`
- `为语音、角色、陪伴型 AI 产品提供统一的桌面入口。`

### Logo 资源

- 使用用户指定的现成 SVG：
  - `https://github.com/XnneHangLab/XnneHangLabTTS/raw/main/assets/imgs/logo.svg`
- 不在 README 中直接引用远程链接
- 需要把该 SVG 下载到当前仓库内，作为本地资源使用

## README 设计

本次新增项目级 `README.md`，结构保持直接、简洁：

1. 项目标题与 logo
2. 项目简介
3. 适用场景
4. 技术栈
5. 当前能力
6. 开发运行方式
7. 项目结构
8. 后续扩展方向

README 中需要同时讲清三层信息：

- 这是模板仓库
- 当前品牌方向叫 `绘心`
- 当前 UI 风格参考绘世启动器，但模板本身不绑定单一业务

## 应用内品牌调整

本次同时修改应用左上角品牌区：

- 去掉当前模板化的彩色方块品牌标记
- 换成新的本地 SVG logo
- 品牌文字直接显示 `绘心`

这次只处理应用内品牌展示，不改页面业务结构。

## 明确不做

为了控制范围，本次不做以下内容：

- 不重做 `src-tauri/icons/` 下的桌面图标包
- 不生成新的多尺寸位图、`.ico`、`.icns`
- 不修改窗口标题、安装包名、Tauri bundle 名称
- 不扩展新的产品页或品牌故事页
- 不把模板仓库重命名为具体产品仓库

## 实施边界

这次代码改动应限制在以下范围内：

- 新增项目级 `README.md`
- 新增本地 logo SVG 资源
- 修改应用左上角品牌区所需的 React / CSS

不应扩散到：

- 启动逻辑
- 设置逻辑
- Tauri command
- 桌面图标生成流程

## 预期结果

完成后应达到：

- 仓库首页一眼能看出“绘心”的品牌方向
- 用户能同时理解它是品牌方向，也是模板仓库
- 应用界面与 README 的品牌表达一致
- 后续如果拆出 `绘心-voice` 等产品，不需要推翻当前模板定位
